Workforce & Utilization Intelligence: User Guide

Welcome to the Workforce & Utilization Intelligence dashboard! This application is designed to ingest standard labor/time-tracking export files and transform them into interactive, actionable insights regarding workforce utilization, non-productive time, non-billable footprint, and resourcing trends.

Security Note: This application runs 100% locally in your web browser. Your data is never uploaded to any external server or cloud database, ensuring complete data privacy and compliance.

1. Getting Started: Data Import

To begin, drag and drop your raw time-tracking Excel file (.xlsx or .xls) into the upload zone, or click to browse your files.

1.1 Expected Column Format

The application expects specific columns to function correctly. However, it features a smart Alias Mapping System that automatically handles common variations in column names. Extra columns (like Resource Primary Role, Resource OBS, etc.) are safely ignored.

Required Columns (or accepted aliases):

1. Date
2. Oracle Code (or Oracle Proj ID)
3. Client Name
4. Project Code
5. Project Description (or Project Name)
6. Resource Name
7. Resource Manager
8. Timesheet Status
9. Week Beginning (Monday)
10. Task Name
11. Hours Billed (or Hours)
12. Type Code

Timesheet Details Report → Allscripts Timesheet detail by resource V3.11

2. Core Business Logic & Calculations

The most important part of this application is how it handles dirty data, calculates capacity, and categorizes time.

2.1 Time Categorization (Prod vs. Non-Prod vs. Non-Bill)

  • Non-Productive The application hardcodes Oracle Code 100002 as the Non-Productive code (e.g., PTO, Bench, internal admin).
  • Non-Billable Any row where the Type Code equals "NON" and the Oracle Code is not 100002. This captures work done for clients or projects that cannot be billed financially.
  • Productive Any remaining rows that do not meet the above two criteria are automatically flagged as Productive.

2.2 The Deduplication Engine

Raw data exports often contain duplicate entries due to corrections and approvals. The system applies a strict hierarchy per Resource and Week:

  • Ignore Invalid: Any row with a status containing Cancelled or Returned is immediately discarded.
  • Hierarchy: Keeps only Adjusted entries if they exist. If none exist, keeps only Posted. Otherwise, it keeps what is available (e.g., Submitted or Open).

2.3 Strict Capacity Logic (The "Trailing Edge" Rule)

Accurately measuring capacity is calculated dynamically based entirely on a user's data presence:

  • Standard Capacity: 40 hours per week.
  • Start/End Date: Bounded by the very first and very last week a resource logs an entry.
  • Internal Gaps: If a resource misses an entry between their start and end date, the system still counts 40 hours of capacity for that gap.
  • The 1-Week Trailing Buffer: If a resource's last entry is exactly one week prior to the global dataset maximum, the system adds 40 hours of capacity for the current week, assuming they are just late submitting.
  • Attrition Cut-off: No capacity is recorded for resources missing two or more consecutive weeks at the end of the global timeline.

2.4 Attrition Analysis Logic

If a resource billed any hours under Manager A in Month X, but has zero billed hours under Manager A in Month X+1, they are counted as +1 Attrition for Manager A in Month X. (The final month is ignored).

3. Navigating the Dashboard

3.1 Executive Summary

KPI Cards: Total Productive Hours, Non-Productive Hours, Calculated Capacity, and Overall Utilization %.

Overall Trends: Line charts showing the ebb and flow of utilization week-over-week and month-over-month.

3.2 Productive Analysis

Focuses strictly on client-facing/productive efforts.

Manager Trends: Compare which managers are driving the highest utilization.

Resource-wise Utilization: A vertical bar chart showing individual utilization. Use the dropdown to sort alphabetically, or by highest/lowest utilization.

3.3 Non-Productive Analysis

Focuses on internal time, bench time, or PTO.

Task Distribution (Donut Chart): Shows exactly where non-productive time is being spent.

Top Resources: Identifies which resources are recording the most non-productive hours.

3.4 Non-Bill Analysis

Focuses on client work that is not yielding revenue.

Smart KPIs: Displays Total Non-Bill Hours, Average Monthly Volume, and the Peak Month.

Comparative Insight Box: Automatically calculates Month-over-Month (MoM) % changes and baseline comparisons (Latest vs. First Month) under the Monthly chart.

Manager Trend Lines: A multi-line chart tracking non-billable volume across top managers over time.

3.5 Resourcing Analysis

Focuses on headcount rather than hours.

Active Resource Count & Attrition: Tracks the number of unique individuals billing time per manager, alongside a stacked attrition chart and YTD table.

Current Resource Count: A bar chart tracking the active headcount by Manager for the most recent week based on capacity.

Client FTE: Converts productive hours logged against a client into a Full-Time Equivalent (FTE) metric against the global timeline to show the actual proportional footprint. The chart displays both the calculated FTE and the total exact hours billed (e.g., 2.6 FTE (104 hrs)).

4. Interactive Features

4.1 Global Cascading Filters

  • Cascading Logic: The filters talk to each other. For example, if you filter by a specific Resource Manager, the Resource Name, Project Code, and Client Name dropdowns will instantly shrink to only show the items associated with that manager.
  • Smart Sorting & Formatting: The Month and Week Beginning filters automatically sort chronologically (e.g., Jan, Feb, Mar) rather than alphabetically. Additionally, all dates across the dashboard, filters, and grids are universally standardized to a clear dd-Mmm-yyyy format (e.g., 02-Feb-2026) to prevent regional confusion.

4.2 Chart Cross-Filtering (Drill-downs)

Every chart on the dashboard is interactive.

  • Filter: Click on any bar, pie slice, or line point to instantly filter the entire dashboard by that specific dimension.
  • Clear: Click that same element again, or click anywhere in the blank background of that chart, to remove the filter.

4.3 Data Grids & Exporting

At the bottom of every tab is a data grid section.

  • Summary Data Grid: Use the Group By: dropdown to pivot your data by Resource, Manager, Task, Project, Client, or Week Beginning. It will calculate exact hours, capacity, and utilization. (Note: When viewing the Non-Bill tab, this grid automatically surfaces the Oracle Code and Client Name).
  • Hierarchical Grouping (Resourcing Tab): When viewing the Summary Grid under the Resourcing Analysis tab, grouping data unlocks a multi-level nested view. For example, grouping by Manager allows you to toggle and drill down into specific Resources, and then further down into specific Clients. (Note: Capacity and Utilization % columns are intentionally hidden in this view to focus strictly on structural hierarchy and billed hours).
  • Expand / Collapse All: When using nested hierarchical grouping, use the quick-action buttons next to the "Group By" dropdown to instantly open or close all data tiers with a single click.
  • Raw Data Grid: Shows the row-by-row granular data. Use the local search box to find specific entries.
  • Smart Export: The blue/green Export buttons will generate a .csv file that respects exactly what is currently visible on your screen (including anything typed into the search boxes).

4.4 Global Navigation & Reporting Controls

Located at the top right of the application navigation bar, these controls are always accessible:

  • Visual Snapshot: Click this button to automatically generate a complete, visual report of your current dashboard. The engine will cycle through all your tabs, take high-resolution screenshots of your currently filtered charts and metrics, and package them into a single, offline HTML file. You can easily email this file or open it later to view the visual report without needing to re-upload the data.
  • Save Session: Click this to download your active session as a lightweight .json file. You can drop this .json file back into the upload zone at any time to instantly restore your exact filters and data state without needing the original Excel file!
  • Built-in User Guide: Click the "User Guide" button to open an interactive overlay of this documentation directly inside the dashboard. This allows you to quickly reference business rules and logic without losing your current filtered session.
  • Reset: Click this to instantly clear all loaded data and filters, allowing you to start fresh with a new Excel file.
